# Definition of Done (DoD)

## 1) Code
- [ ] Respect des conventions de nommage et architecture.
- [ ] Pas de code mort / TODO critique non traité.
- [ ] Complexité maîtrisée (méthodes courtes, responsabilité claire).

## 2) Qualité
- [ ] Tests unitaires ajoutés/ajustés.
- [ ] Tests d’intégration sur cas critiques.
- [ ] Régression vérifiée sur flux impactés.

## 3) API/Contrats
- [ ] Contrats request/response stables.
- [ ] Erreurs standardisées.
- [ ] Documentation Swagger à jour.

## 4) Sécurité
- [ ] Contrôles authN/authZ vérifiés.
- [ ] Validation entrée complète.
- [ ] Aucune donnée sensible exposée.

## 5) Observabilité
- [ ] Logs structurés pertinents.
- [ ] Trace/correlation id disponible.
- [ ] Health check si nouveau composant.

## 6) Livraison
- [ ] PR relue et approuvée.
- [ ] CI verte.
- [ ] Feature flags/config env documentés.
- [ ] Notes de release mises à jour.
