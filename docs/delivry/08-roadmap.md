# Roadmap Template (Lead Dev)

## Horizon
- H1: MVP
- H2: Stabilisation
- H3: Scalabilité / industrialisation

## Sprint 0 — Cadrage & setup
- Vision produit validée
- Architecture validée
- Conventions codées dans lint/CI
- CI/CD initiale

## Sprint 1 — Socle vertical
- AuthN/AuthZ
- 1 module métier complet de référence
- Observabilité minimale (logs, health)

## Sprint 2 — Core features
- 2 à 3 features prioritaires
- Qualité API + tests intégration
- UX states cohérents (loading/error/empty)

## Sprint 3 — Fiabilisation
- Performance listes et endpoints critiques
- Hardening sécurité
- Couverture de tests augmentée

## Sprint 4 — Pré-prod
- UAT
- correction defects bloquants
- documentation opérationnelle

## Gouvernance hebdo
- KPI engineering: lead time, cycle time, defect escape rate.
- KPI produit: activation, time-to-value, adoption.

## Definition of Ready (DoR)
- User story claire + critères Given/When/Then.
- Impact technique identifié.
- Dépendances explicites.
